# オンライン会議用 AI コパイロット開発

## プロダクトのコンセプト

オンライン会議に特化した、既存ツールを凌駕するシンプルかつ強力なデスクトップアプリ。
商談中にバレずに使う「透明なカンペ」として、営業・経営層の商談勝率を上げることに特化する。

アプリ名：**Invisible Teleprompter（透明なカンペ）**

ブラウザ拡張機能ではなく、PC（Windows/Mac）で動く超軽量なデスクトップ常駐アプリ。
これにより Google Meet（ブラウザ）、Zoom（アプリ）、Teams（アプリ）のすべてにシームレスに対応できる。

## 既存ツールの弱点

| ツール例 | 問題点 |
|---------|--------|
| Limitless | 機能が多すぎて画面が邪魔、処理が重い |
| Otter | Bot が参加して相手にバレる |
| Sider | ブラウザでしか動かない（Zoom デスクトップ版で使えない） |

## 3つのコア機能

### 1. 透明ウィジェット UI（物理的ステルス）

画面上部の中央に、背景が透明な数行のテキストだけを浮かび上がらせる。

- ユーザーはカメラ目線を維持したまま AI の提案（箇条書き）を読める
- 相手からは「まっすぐ目を見て、淀みなく的確な回答をしている」ようにしか見えない

### 2. 領域指定による軽量マルチモーダル

アプリ起動時に「相手の画面共有が表示されるエリア」だけをマウスでドラッグして指定（OBS のウィンドウキャプチャのようなイメージ）。

- AI は「その枠内の画像（資料）」と「PC の内部音声（相手の声）」だけを数秒ごとにセットで処理
- 画面全体を録画する Limitless に比べ、負荷・プライバシーリスクともに低い

### 3. Push-to-Think（思考ボタン）

AI が常時テキストを出し続けると、ユーザーは読むことに必死になり商談に集中できない。
普段はバックグラウンドで議事録の構造化のみを静かに進め、ユーザーがショートカットを押した瞬間だけ「直前 30 秒の音声＋現在のスライド画像」から 3 つの切り返しトークを表示する。

- ①論破
- ②同調からの提案
- ③質問返し

各 20 文字程度で透明ウィジェットに表示する。

## システムアーキテクチャ（シンプル構成）

| レイヤー | 技術選定 | 理由 |
|---------|---------|------|
| デスクトップ UI | Electron または Tauri | 透過ウィンドウ（Always on Top）が実装しやすい |
| 音声取得 | PC のループバック音源をキャプチャ | スピーカー出力を取得 |
| 映像取得 | 指定矩形領域のスクリーンショット（数秒に 1 回） | 軽量・プライバシー保護 |
| 音声認識 | Whisper API または Deepgram | 超低遅延テキスト化 |
| AI 推論 | Gemini 1.5 Flash 等のマルチモーダル API | 画像＋テキストの高速処理 |

## 競合優位性

「議事録を残すこと」を主目的とした既存ツールは山ほどあるが、「今この瞬間の目の前の相手との交渉に勝つこと」に特化したリアルタイムのコパイロットはまだ決定的な覇者がいない。

「すべてを記録する重いツール」ではなく、「商談中の数十分だけ自分の知能をブーストする軽いツール」という見せ方にすることで、セールスマンや経営層に深く刺さる。
