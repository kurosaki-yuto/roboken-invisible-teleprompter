# SaaS 開発計画：多言語・課金・無料化

収益性と拡張性を兼ね備えたグローバル SaaS として完成させるための設計・実装プラン。
「構築済みのコンポーネント」を活用し、Cursor での開発速度を落とさない構成にする。

---

## 1. グローバル対応・多言語化戦略（i18n）

UI の 50 ヶ国語対応と自動切り替えを最小限の工数で実装する。

**技術選定**: `i18next` + `react-i18next`

### 言語判定ロジック（優先順）

1. ユーザーのブラウザ設定（`navigator.language`）を優先
2. サインアップ済みの場合は、ユーザープロフィールに保存された言語設定を適用
3. IP アドレスによる位置判定（Cloudflare 等の CDN 層、または Stripe の提供データ）で通貨・デフォルト言語を補助的に判定

### 翻訳資産の管理

50 ヶ国語の文言管理は JSON ファイル（`ja.json`, `en.json` 等）を用意し、Cursor の全ファイル編集機能で一括翻訳・生成する。

## 2. 高収益を実現する料金プランと原価計算

主な原価: リアルタイム音声認識（Deepgram）+ LLM（Gemini 1.5 Flash）の API 利用料

**商談 1 時間の API 原価目安**: 約 $0.5〜$1.0

| プラン | 月額 | 制限 | 月間原価（上限使用時） | 利益率 |
|--------|------|------|----------------------|--------|
| Free | 無料 | 月間 30 分 | 〜$0.5 | — |
| Standard | $29 | 月間 10 時間 | $5〜$10 | 70〜80% |
| Business | $79 | 無制限・チーム管理・高度な議事録検索 | 変動 | 〜70% |

Standard プランで月間 10 時間フルに使われても、原価は約 $5〜$10 程度。利益率 70〜80% を維持できる。

---

## 3. 課金・認証・内部利用フラグの実装

### 決済基盤: Stripe Checkout

独自の決済画面は作らず、Stripe が用意する決済用ページへリダイレクトさせる。

### 認証基盤: Clerk または Firebase Auth

ユーザー属性（Metadata）に `role: "admin"` または `is_internal: true` フラグを持たせる。

### 関係者無料化ロジック

```
if メールアドレスのドメイン === "@your-company.com":
    → plan: 'pro_internal'（無料・無制限）を自動付与
elif DB上の is_free_user === true:
    → Stripe の課金チェックをバイパス
else:
    → Stripe の支払い完了（Webhook 受信）後にのみ Pro 機能を解放
```

## 4. 必須ページとリーガル対応

Cursor にベースとなる文章を生成させ、専門的な構成にする。

| ページ | 内容 |
|--------|------|
| LP / 説明ページ | 「オンライン会議の勝率を上げる AI 参謀」としてのベネフィットを強調 |
| FAQ | 「相手にバレませんか？」「セキュリティ（録音データ）の扱いは？」への回答 |
| 利用規約 | AI によるデータ処理とプライバシー保護について明記 |
| プライバシーポリシー | AI によるデータ処理とプライバシー保護について明記 |
| 特定商取引法に基づく表記 | 日本国内展開に必須 |

標準テンプレートを Cursor に読み込ませ、サービス名・運営会社名を置換して生成する。
